## 🎯 Project Overview

In traditional software development, sprint planning relies heavily on human estimation, which often leads to inaccurate deadlines, bottlenecks, and developer burnout. This system removes the guesswork by predicting task durations based on historical complexity data and utilizing an evolutionary algorithm to find the mathematically optimal task distribution among a multi-tiered engineering team.

### 🧠 Core Features & Architecture

*   **Predictive ML Pipeline:** Utilizes a custom-trained Scikit-Learn regressor model tracked via Validation Loss (MAE) metrics to predict realistic execution times for backlog items.
*   **Combinatorial Optimization Engine:** Implements an Elitist Genetic Algorithm to solve the NP-Hard scheduling problem across thousands of possible permutations in milliseconds.
*   **Directed Acyclic Graph (DAG) Timeline Scheduling:** Dynamically accounts for task sequence dependencies, adjusting timelines sequentially.
*   **Resource & Burnout Safeguards:** Enforces strict daily workload thresholds and skill-level ceilings (e.g., preventing a Junior Developer from being assigned a critical architectural task).

## 🖥️ Live Telemetry Dashboard Interface

The application features a sleek, cyberpunk-styled DevOps interface built with Streamlit, enabling project managers and stakeholders to interact with constraints in real time.

1. **Core Engine Settings:** Fine-tune GA parameters such as Population Size, Generations, and Mutation Probability.
2. **Execution Telemetry:** Trigger the optimization pipeline to see live generational search spaces converge onto the lowest-cost path.
3. **Resource Heatmaps:** Instantly visualize developer workloads against safety thresholds to quickly detect overload risks.
